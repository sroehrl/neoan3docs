Unicore Singleton

Tapping into the power of the Unicore

The singleton is initiated in a route component by executing the following with a particular frame:

$this->uni('test')

Useful methods

There are multiple methods of interest that can be chained to the singleton. 



        namespace Neoan3\Components;

        use Neoan3\Core\Unicore;

        class UnicoreSingleton extends Unicore
        {
            function init()
            {
                $this->uni('docs')
                     ->hook('main','unicoreSingleton')
                     ->output();
            }

        }

Hook

The hook helps to indicate what part of the frame to target and what view to use. Optionally you can even take in an array of substitutions declared in the view by double-curly braces. Example: 


$this->uni->hook('main', 'test');
//src/test/test.view.html will hook into the frame's main-hook

Typically, the view passed as a parameter will be the view.html that is local to the ctrl.php.

addHead

addHead has two parameters "what" and "obj". "what" indicates what you want to add to the head tag of the html. For instance, If you pass "title", you can pass an "obj" that will indicate what appears in the title tag. 


->addHead('title', isset($this->content['name']) ? $this->content['name'] : 'Overview')

addController

addController is a method that by it's namesake allows you to add a controller. This is useful when there is a function in another ctrl.php file that you would use in the current one. The parameter is simple and takes only the name of the controller. 


->addController('profile')

Callback

Callback adds the ability to run a function simply by adding the name of the function as a parameter. 


->callback($this, 'getContext')
//parameters(context, function)
...
...
function getContext()
{
    //do something
}

Output

Output simply compiles the files & responds (echoes) to the client. The parameter isn't necessary but you can add a string or array of pre-compiling function(s). Typically this is also at the end of a singleton. 


->output();
