Templating

Creating the view

Whether you use the neoan3 cli tool, or you create your own component using your IDE, a view for the user is important. That's where templating comes in, and with neoan3 it's easy to create and route to your view.

Creating a view with neoan3 cli tool

Views are easy to create with the neoan3 cli tool. After opening your favorite terminal, you simply type:

neoan3 new component home

The cli tool will ask the following questions:

? This component mainly serves as: (Use arrow keys) >Route component API endpoint Custom Element

Select Route component

? Do you want to create a view (Y/n)

Type y for yes

? do you want to use a frame? (Y/n)

NOTE: For more details about Frames please review Frames

For now we will assume that we have a frame called Test Type y for yes

? Which frame are you using? (Use arrow keys) > test

Hit Enter on your keyboard to select that frame

writing... using html-template... Component test created

Success! You have your first route component.

Creating a view without neoan3 cli tool

There are some considerations to be made when creating them without the neoan3 cli tool. All views are contained in the same folder as their relevant component. In that regard a 'home' folder could conceivably have the following files:

1. Home.ctrl.php

2. home.view.html

As you see, the name of the view should have the following format:

{{componentName}}.view.html

This guarantees that the routing, which is taken care of by the ctrl.php file, will direct a user to the appropriate view.

Routing to a view

Routing to a view is very simple in neoan3, especially using the neaon3 cli tool. The Route Component section will detail further what a Route component is.
