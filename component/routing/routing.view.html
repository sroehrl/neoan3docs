Routing

General routing

Neoan3 has made the process of routing easy for fast development. Whether you're creating a route for a web page, an API or to include a node & custom elements neoan3 has a strategy.

Principals of routing

As Neoan3 is a PHP based framework, routing is handled through the PHP controller in a target component's folder. As an example, the following route component named home results in routing for a web page. Note: While the following is a basic example of routing, please review the full explanation of routing to a web page. 

    
    home (component folder)
        |
        Home.ctrl.php
        home.view.html

As you see above, the home.view.html has a companion Home.ctrl.php which handles routing. If you're using the neoan3 cli tool this step is easy, however let us take a look at the default routing necessary to be able to navigate to our home.view.html in a web browser. 

        
            namespace Neoan3\Components;

            use Neoan3\Core\Unicore;

            class Home extends Unicore
            {
                function init()
                {
                    $this->uni('docs')->hook('main','home')->output();
                }

            }

While there is a lot to unpack from the above the essential thing to understand is that the routing is placing the 'home' view specified in the hook into 'main'. Neoan3 will look for a local home.view.html to refer to for routing. This would yield a slug to your localhost if you are running this locally.

http://localhost/myProject/home
